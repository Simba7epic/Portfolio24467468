CVE-2026-26113: Microsoft Office Remote Code Execution Vulnerability
What It Is
CVE-2026-26113 is an untrusted pointer dereference vulnerability in Microsoft Office that allows an unauthorized attacker to execute code locally. 
CVE

CVE-2026-26113 involves improper handling of embedded objects in Office documents. The vulnerability exists in how Office processes OLE (Object Linking and Embedding) objects, allowing attackers to bypass security checks and execute code with the privileges of the current user. 
Windows News

Disclosure & Severity
CVE-2026-26110 and CVE-2026-26113 are a pair of Office bugs where the Preview Pane is an exploit vector, disclosed as part of Microsoft's March 2026 Patch Tuesday. 
Zero Day Initiative

Both CVEs received CVSSv3 scores of 8.4 and were rated as critical. A local, unauthenticated attacker could exploit these vulnerabilities to achieve local code execution. Microsoft notes that the preview pane is an attack vector for these flaws, and both CVEs were assessed as "Exploitation Less Likely." 
Security Boulevard

Affected Products
Both vulnerabilities affect Office 2016, Office 2019, Office 2021, and Microsoft 365 Apps for Enterprise. Microsoft has confirmed that Office for Mac and mobile versions are not affected. Windows Server versions running Office are vulnerable if the Desktop Experience feature is installed. 
Windows News

How Exploitation Works
This flaw is particularly concerning because it can be triggered through multiple attack vectors, including email attachments and downloaded documents. When Office processes certain file formats through the preview pane, improper memory handling allows attackers to overwrite critical memory addresses, which can be weaponized to redirect program execution to attacker-controlled code. 
Windows News
Windows News

Successful exploitation grants attackers the same privileges as the logged-in user. In corporate environments where users often have elevated permissions, this could lead to complete system compromise, data exfiltration, and lateral movement across networks. 
Windows News

Mitigation Strategies
For System Administrators
1. Apply the Patch Immediately Apply Microsoft's security updates via Windows Update, Microsoft Update Catalog, or your enterprise management tools (SCCM/ConfigMgr, WSUS, Intune). For Click-to-Run installs, ensure Microsoft 365 Apps auto-update is enabled or run updates from within an Office application via File → Account → Update Options → Update Now. 
Windows Forum

2. Disable the Preview Pane Since the preview pane is a confirmed attack vector, disabling it in Windows Explorer and Outlook significantly reduces the attack surface on systems that cannot be patched immediately.

3. Enable Protected View Enable Office Protected View for files from the internet, and configure Microsoft Defender to block Office macros from untrusted sources. 
Windows News

4. Verify Patch Deployment Use vulnerability scanning tools and inventory to confirm affected Office versions were updated. Many vendors published Patch Tuesday advisories summarizing which KBs and builds remediate each CVE — consume those vendor mappings and correlate with your asset inventory. 
Windows Forum

5. Isolate Unpatched Systems For servers or systems that cannot be patched immediately, implement temporary mitigations and create a prioritized patch schedule, treating any systems that cannot be patched as high-priority assets for isolation and monitoring. 
Windows Forum

For Developers
6. Enforce Least Privilege Since the vulnerability executes with the current user's privileges, ensuring application and user accounts run with the minimum necessary permissions limits the blast radius of any successful exploit.

7. Validate and Sanitize OLE/Embedded Object Handling If your application processes or renders Office documents (e.g., via Office interop or third-party libraries), audit how OLE objects and embedded content are handled. Avoid rendering untrusted documents in full-trust contexts.

8. Integrate Vulnerability Scanning into CI/CD Incorporate tools like Tenable, Qualys, or Microsoft Defender Vulnerability Management into your pipeline to catch dependency and platform vulnerabilities before they reach production.

9. User Awareness Train users not to preview or open Office documents from untrusted email sources, and to treat unexpected file attachments with suspicion — even before previewing them, since the preview pane itself is an attack vector here.

Bottom line: This is a patched vulnerability — the fix is available now. The most critical action for both developers and sysadmins is to apply the March 2026 Patch Tuesday update without delay, and to disable the Office Preview Pane as a temporary control on any systems where immediate patching isn't feasible.
